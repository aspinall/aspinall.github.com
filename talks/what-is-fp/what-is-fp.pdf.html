What is Functional Programming?

There is a lot of misinformation and misinterpretation around this question. Let us start by defining some of the things that functional programming is not!

It is not about type systems. The question of strong or weak, static or dynamic, applies to both functional and imperative programming.

It is not about lazy or strict evaluation either!

It is not about higher order functions.

And it is certainly not about monads!

To find out what functional programming is really about, we need to go back in time…

History Lesson

In the 1930s at Princeton university, there were a number of people trying to answer questions related to computation, such as “if we had infinite computing power, what problems would we be able to solve?”

One of those people was Alonzo Church. He developed a formal system called lambda calculus, essentially a programming language for a machine with infinite power. It was based on functions that took other functions as parameters and returned functions. Using this he was able to provide conclusive answers to the many of the computation questions.

Another was Alan Turing, who independently developed a different formal system now known as the Turing machine, and using it he came to similar conclusions as Alonzo Church.

It was later shown that Turing machines and lambda calculus were equivalent in power.

The third person was John von Neumann. In 1949 the EDVAC (Electronic Discrete Variable Automatic Computer) was unveiled and was the first example of von Neumanns architecture, effectively a real world implementation of a Turing machine.

In 1958, John McCarthy (a Princeton graduate) unveiled LISP (LISt Processing language), an implementation of lambda calculus that worked on von Neumann computers.

So… What is Functional Programming?

Functional programming is a practical implementation of lambda calculus. (Practical in the sense that not all lambda calculus ideas can be implemented under physical limitations!)

At the core of functional programming is the function (unsurprisingly). They are used for almost everything.

A function takes one or more arguments and returns a result.

A pure function does not use anything outside of its own scope, which means it is guaranteed to return the same result given the same argument values.

In functional programming variables are simply aliases for expressions. They can only be assigned to once and cannot be modified. Since they are immutable, they are really variables in the mathematical sense rather than the imperative programming sense you might be used to.

OK… Why should we consider Functional Programming?

The combination of pure functions and immutable data leads to referential transparency, which means that the result of the function can be calculated at any time and substituted for the function call without affecting the result of the program. This is because the function does not rely on, or affect, any external state, only its arguments.

This has huge implications for testing your code. You can unit test every function just by calling it with different arguments.

Similarly, debugging a function that isn’t behaving the way you expect is much easier because its behaviour is not dependent on code that has been executed before it.

No shared state or mutable data means that your functions are automatically thread safe too!

Hang on a minute… no shared state? How can you possible write a meaningful program with no shared state?

OK, so there is state, but its not state as we know it! State is not held in variables, it is held in function parameters on the stack. To modify the state, you call functions (often recursively) passing different values.

But don’t I need to be using a Functional Programming language?

In a word, no!

In fact, it’s debatable whether there is any such thing as a “functional programming language”. Remember that it was proven that lambda calculus and Turing machines were equivalent in power? That means that whatever you can implement in one, you can implement in the other.

That said, some languages lend themselves to functional programming better than others:

Better suited to functional programming: LISP, ML, Haskell, Erlang

Less suited to functional programming: Java, C#, C, C++, Ruby, Python

Somewhere in the middle: Scala, F#

So what makes a language more suitable for functional programming?

Language Traits That Aide Functional Programming

  • Declarative – describe what rather than how
  • Functions are first class values – they can be passed around
  • Higher order functions – functions that operate on other functions
  • Immutable data – cannot modify data once initialized
  • Recursive loops – instead of iteration (for, while, repeat etc.)
  • Algebraic data types – simple containers that can be recursively defined
  • Pattern matching – on primitive values and algebraic data types

FizzBuzz

This is a simple coding exercise I used to get candidates to complete during telephone interviews using Google Docs. They had five minutes to provide a solution and the sole purpose of the exercise was to check that they could actually code (not to judge how well they coded)!

Write a program that prints the numbers from 1 to 100 except:

  • for each number that is a multiple of 3, print “Fizz” instead of the number
  • for each number that is a multilpe of 5, print “Buzz” instead of the number
  • for each number that is a multiple of 3 and 5, print “FizzBuzz” instead of the number

Incidentally, you would be amazed at the number of people applying for senior development positions who could not complete this exercise!

Typical “FizzBuzz” Answer in Java

public class FizzBuzz {
	public static void main(String[] args) {
		for (int i = 1; i <= 100; i++)
			if ((i % 3 == 0) && (i % 5 ==0))
				System.out.println("FizzBuzz");
			else if (i % 3 == 0)
				System.out.println("Fizz");
			else if (i % 5 == 0)
				System.out.println("Buzz");
			else
				System.out.println(i);
	}
}

A Better “FizzBuzz” Answer

import java.util.*;

public class BetterFizzBuzz {
	static List<String> fizzBuzz(int upperBound) {
		ArrayList<String> results = new ArrayList<String>();
		for (int i = 1; i <= upperBound; i++)
			if ((i % 3 == 0) && (i % 5 ==0))
				results.add("FizzBuzz");
			else if (i % 3 == 0)
				results.add("Fizz");
			else if (i % 5 == 0)
				results.add("Buzz");
			else
				results.add(String.valueOf(i));
		return results;
	}

	public static void main(String[] args) {
		for(String line: fizzBuzz(100))
			System.out.println(line);
	}
}

A More Functional “Fizzbuzz” Answer in Java

import java.util.*;

public class FunctionalFizzBuzz {
	static boolean isMultipleOf3(final int num) { return num % 3 == 0; }

	static boolean isMultipleOf5(final int num) { return num % 5 == 0; }

	static String fizzBuzzOrNumber(final int num) {
		if (isMultipleOf3(num) && isMultipleOf5(num))
			return "FizzBuzz";
		else if (isMultipleOf3(num))
			return "Fizz";
		else if (isMultipleOf5(num))
			return "Buzz";
		else
			return String.valueOf(num);
	}

	static List<String> fizzBuzz(final int upperBound) {
		ArrayList<String> results = new ArrayList<String>();
		for (int i = 1; i <= upperBound; i++)
			results.add(fizzBuzzOrNumber(i));
		return Collections.unmodifiableList(results);
	}

	public static void main(String[] args) {
		for(String line: fizzBuzz(100))
			System.out.println(line);
	}
}

A Functional “FizzBuzz” Answer in Haskell

Note that I wrote this, so it may not be the most idiomatic Haskell example you will find!

import Control.Monad

multipleOf3 num = num `mod` 3 == 0

multipleOf5 num = num `mod` 5 == 0

fizzBuzzOrNumber num =
	if multipleOf3 num && multipleOf5 num then "FizzBuzz" else
		if multipleOf3 num then "Fizz" else
			if multipleOf5 num then "Buzz"
				else show num

fizzBuzz upperBound = map fizzBuzzOrNumber [1..upperBound]

main = forM_ (fizzBuzz 100) putStrLn
